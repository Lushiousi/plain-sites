Валидный код - это код, который соответствует стандартам.

На валидность можно проверить HTML, CSS, всяческие микроразметки и другое. Здесь речь пойдет про валидность в HTML.

  • Валидный код необязателен, но количество ошибок должно быть минимальным, иначе сайт не будет кроссбраузерным. Валидность кода нужна в прежде всего для того, чтобы сайт отображался правильно во всех браузерах.
  • Поисковые роботы "разговаривают" с сайтом на языке HTML, поэтому важно отдавать четко и ясно контент на сайте со всеми "закрытыми тегами" и прочее.
  • Валидность HTML влияет на SEO, но довольно незначительно (если, конечно, не сотни, а то и тысячи ошибок). Рекомендуется почитать интересные наблюдения Деваки "Влияние качества HTML на их ранжирование".
  • Когда люди делают код валидным, то заодно находят и исправляют свои глупые ошибки (повторение тегов, пропущенная буква и т.п.).
  • Не стоит надрываться, если какую-то ошибку сложно исправить, либо ее исправление принесет вред функциональности сайта. Главное, чтобы было удобно пользователю.

В каждой ошибке есть подсказка - это номер строки в исходном коде странице, а из нее уже можно определить примерно в каком файле темы расположена данная строка. Исходный код страницы смотрим с помощью CTRL+U (в основных браузерах).

Перед тем, как приступить к работе, нужно сделать резервную копию шаблона сайта.

Также для упрощения нахождения ошибок в исходном коде, можно использовать HTML валидатор для Mozilla Firefox. Установив его, перейдя в исходный код страницы, видно те же самые ошибки, что указывает сервис validator.w3.org. Кликнув по названию ошибки (в левом нижнем углу), автоматически перебросит на ту строчку, где находится данный невалидный код.
